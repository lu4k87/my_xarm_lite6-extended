ROS 2 Gesamtübersicht (BOOTCAMP)

1. ROS 2 Workspace & Pakete

Ein Workspace ist das Verzeichnis, in dem ROS 2-Code entwickelt und gebaut wird. Das grundlegende ROS-System (z. B. in /opt/ros/humble) bildet das Underlay. Der eigene, darüber geladene Workspace ist das Overlay.

  • src/: Hier liegen die Quellcodes (Packages) des Benutzers.
  • build/: Temporäre Dateien, die während des Build-Prozesses entstehen.
  • install/: Kompilierte Dateien, Skripte und Executables (dieser Ordner wird per source in die Umgebung geladen).
  • log/: Log-Dateien des Build-Prozesses zur Fehleranalyse.

Ein Paket (Package) ist die organisatorische Grundeinheit in ROS 2. Es bündelt Code (Nodes), Bibliotheken, Konfigurationsdateien oder Launch-Files. Ein typisches Python-Paket (ament_python) besteht aus:

  • package.xml: Enthält Metadaten (Name, Version, Autor) und Abhängigkeiten (Dependencies), die das System zum Ausführen benötigt (z.B. rclpy, std_msgs). Wird auch von rosdep gelesen.
  • setup.py: Das Bau-Skript. Hier werden Launch-Files registriert (data_files) und die Entry Points definiert. Ein Entry Point sagt dem System genau, welcher Aufrufname im Terminal zu welcher .py-Datei und welcher Funktion darin führt.
  • setup.cfg: Teilt dem System mit, wo die ausführbaren Dateien abgelegt werden sollen.
  • <paket_name>/: Der Ordner, der die eigentlichen Python-Skripte (die Nodes) enthält.
Aktion Befehl(e) / Ausführung
Workspace Ordner anlegen mkdir -p ~/ros2_ws/src && cd ~/ros2_ws

2. ROS 2 Grundbegriffe & Architektur

A. Der Node (Knoten)

Ein Node ist ein einzelner Prozess, der Berechnungen durchführt und über eine Client-Bibliothek mit anderen Nodes innerhalb des ROS-Graphen kommuniziert.

  • Vorteile: Fehlertoleranz (isoliert bei Abstürzen), Modularität, Wiederverwendbarkeit. Sprachunabhängig (C++ Nodes können mit Python Nodes kommunizieren).
  • Best Practice: Ein Node sollte genau eine logische Aufgabe erfüllen.

B. Wichtige Konzepte & Mechanismen

Diese Bausteine definieren, wie Nodes strukturiert sind und wie sie im Hintergrund arbeiten:

  • Namespace: Ein Präfix zur Gruppierung von Nodes und Topics (z. B. /turtle1/cmd_vel). Verhindert Namenskonflikte in komplexen Systemen.
  • Spin: Eine Endlosschleife, die den Node aktiv hält, um auf Daten und Ereignisse zu reagieren.
  • Callback: Eine Funktion, die bei einem bestimmten Ereignis (z. B. beim Empfang einer Nachricht) automatisch ausgelöst wird.
  • Executors & Callback Groups: Der Mechanismus "unter der Haube", der entscheidet, wann welcher Callback ausgeführt wird (z. B. Single-Threaded vs. Multi-Threaded).
  • Future: Ein Platzhalter-Objekt für das Ergebnis einer asynchronen Operation, das erst zu einem späteren Zeitpunkt verfügbar ist.
  • Synchron vs. Asynchron: Synchron blockiert den laufenden Prozess, bis eine Antwort kommt. Asynchron lässt den Prozess weiterlaufen, während auf die Antwort gewartet wird.

Code-Beispiel: Minimaler Node ohne Klasse (bare.py)

import rclpy

def main(args=None) -> None:
    rclpy.init(args=args)
    node = rclpy.create_node("bare_node")
    print("Hello World !!!")
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == "__main__":
    main()

Code-Beispiel: Node mit Klasse und Timer (class.py)

import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self) -> None:
        super().__init__("my_node")
        self.count = 0
        self.create_timer(0.5, self.my_callback)

    def my_callback(self):
        self.get_logger().info(f"Hello: {self.count}")
        self.count += 1

def main(args=None) -> None:
    rclpy.init(args=args)
    my_node = MyNode()
    rclpy.spin(my_node)
    rclpy.shutdown()

if __name__ == "__main__":
    main()

3. ROS 2 Interfaces & Code-Beispiele

Interfaces definieren die Struktur der ausgetauschten Daten.

A. Topics (.msg Dateien) - Asynchron

Kontinuierliche Datenströme (Viele-zu-Viele).

Code-Beispiel: Publisher (publisher_msg.py)

import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class PublisherNode(Node):
    def __init__(self) -> None:
        super().__init__("publisher_msg_node")
        self.__pub = self.create_publisher(String, "/message_topic_name", 10)
        self.create_timer(0.5, self.pub_message)

    def pub_message(self): 
        msg = String()
        msg.data = "testnachricht....."
        self.__pub.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')

def main(args=None) -> None:
    rclpy.init(args=args)
    node = PublisherNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

B. Services (.srv Dateien) - Synchron

Remote Procedure Calls (Request / Response) für kurze Abfragen.

Struktur einer Service-Datei:

string str
---
string str

Code-Beispiel: Service Server (server.py)

import math
import rclpy
from rclpy.node import Node
from demo_interfaces.srv import EuclidianDistance

class ServerNode(Node):
    def __init__(self) -> None:
        super().__init__("server_node")
        self.create_service(EuclidianDistance, "/EuclidianDistance", self.compute_distance)

    def compute_distance(self, request, response):
        self.get_logger().info("Got a request...")
        dx = request.target.x - request.origin.x
        dy = request.target.y - request.origin.y
        dz = request.target.z - request.origin.z
        response.distance = math.sqrt(dx**2 + dy**2 + dz**2)
        return response

Code-Beispiel: Service Client (my_client.py)

import rclpy
from rclpy.node import Node
from demo_interfaces.srv import EuclidianDistance
from rclpy.task import Future

class ClientNode(Node):
    def __init__(self) -> None:
        super().__init__("client_node")
        self.client = self.create_client(EuclidianDistance, "/EuclidianDistance")
        while not self.client.wait_for_service(timeout_sec=3.0):
            self.get_logger().info("Waiting for Service....")
        self.__request = EuclidianDistance.Request()

    def send_request(self) -> Future:
        self.__request.origin.x = 0.0
        self.__request.origin.y = 0.0
        self.__request.origin.z = 0.0
        self.__request.target.x = 1.0
        self.__request.target.y = 1.0
        self.__request.target.z = 1.0
        return self.client.call_async(self.__request)

def main(args=None) -> None:
    rclpy.init(args=args)
    node = ClientNode()
    future = node.send_request()
    rclpy.spin_until_future_complete(node, future)
    response = future.result()
    node.get_logger().info(f"Got the result: {response.distance}")
    rclpy.shutdown()

C. Actions (.action Dateien) - Asynchron

Langanhaltende Prozesse. Bestehen aus: Goal (Ziel), Feedback (Fortschritt), Result (Endergebnis). Können abgebrochen werden (preemptable).

Struktur einer Action-Datei:

<request_type> <request_fieldname>
---
<response_type> <response_fieldname>
---
<feedback_type> <feedback_fieldname>

Code-Beispiel: Action Server (action_server.py)

import rclpy
from rclpy.node import Node
from demo_interfaces.action import DriveDistance
from rclpy.action import ActionServer, GoalResponse, CancelResponse
import time

class ActionServerNode(Node):
    def __init__(self) -> None:
        super().__init__("action_server")
        self.action_server = ActionServer(
            self, DriveDistance, "/DriveDistanceAction",
            goal_callback=self.goal_callback,
            cancel_callback=self.cancel_callback,
            execute_callback=self.execute_callback)

    def goal_callback(self, goal_request):
        if goal_request.goal.x == 0 and goal_request.goal.y == 0 and goal_request.goal.z == 0:
            return GoalResponse.REJECT
        return GoalResponse.ACCEPT
    
    def cancel_callback(self, goal_handle):
        return CancelResponse.ACCEPT
    
    def execute_callback(self, goal_handle):
        feedback_msg = DriveDistance.Feedback()
        for i in range(1, 11):
            if goal_handle.is_cancel_requested:
                goal_handle.canceled()
                return DriveDistance.Result()
            feedback_msg.remaining_distance_m = float(11 - i)
            goal_handle.publish_feedback(feedback_msg)
            time.sleep(0.5)
        goal_handle.succeed()
        result = DriveDistance.Result()
        result.success = True
        return result

Code-Beispiel: Action Client (action_client.py)

import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from demo_interfaces.action import DriveDistance

class ActionClientNode(Node):
    def __init__(self):
        super().__init__("action_client")
        self._action_client = ActionClient(self, DriveDistance, "/DriveDistanceAction")

    def send_goal(self, x: float, y: float, z: float):
        target_location = DriveDistance.Goal()
        target_location.goal.x, target_location.goal.y, target_location.goal.z = float(x), float(y), float(z)
        while not self._action_client.wait_for_server(timeout_sec=3.0):
            pass
        self._send_goal_future = self._action_client.send_goal_async(
            target_location, feedback_callback=self.updates_from_server)
        self._send_goal_future.add_done_callback(self.goal_response_callback)

    def goal_response_callback(self, future):
        goal_handle = future.result()
        if not goal_handle.accepted:
            return
        self._get_result_future = goal_handle.get_result_async()
        self._get_result_future.add_done_callback(self.get_result_callback)

    def get_result_callback(self, future):
        result = future.result().result
        self.get_logger().info(f"FERTIG! Ergebnis: {result.success}")
        rclpy.shutdown()

    def updates_from_server(self, feedback_msg):
        actual_feedback = feedback_msg.feedback
        self.get_logger().info(f"FEEDBACK: Verbleibende Distanz: {actual_feedback.remaining_distance_m} m")

5. ROS 2 Netzwerk, Middleware (DDS) & QoS

Middleware: DDS (Data Distribution Service)

DDS ist das Rückgrat der ROS 2 Kommunikation.

  • FastDDS (eProsima): Die Standard-Middleware in vielen ROS 2 Versionen.
  • CycloneDDS (Eclipse): Eine beliebte Alternative (oft stabiler in WLAN-Netzwerken).
  • Wechseln der Middleware:
# Zu CycloneDDS wechseln (muss installiert sein: ros-<distro>-rmw-cyclonedds-cpp)
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp

# Zurück zu FastDDS wechseln
export RMW_IMPLEMENTATION=rmw_fastrtps_cpp

QoS (Quality of Service)

Regelwerk für die Datenübertragung bei instabilen Netzwerken (z. B. WLAN, LTE).

  • Reliability: Reliable (garantiert Zustellung) vs. Best Effort (sendet einmal).
  • History: Keep Last (speichert N Nachrichten) vs. Keep All (speichert alle).
  • Durability: Volatile (keine alten Daten) vs. Transient Local (Speichert für Late-Joiner).
  • Telemetrie: VPNs (OpenVPN/WireGuard) oder Dronebridge-radio empfohlen.

Code-Beispiel: Publisher/Subscriber mit QoS-Profil (qos_pub.py / qos_sub.py)

from rclpy.qos import QoSProfile, QoSHistoryPolicy, QoSReliabilityPolicy, QoSDurabilityPolicy

my_qos = QoSProfile(
    history = QoSHistoryPolicy.KEEP_LAST,
    depth = 10,
    reliability = QoSReliabilityPolicy.BEST_EFFORT,
    durability = QoSDurabilityPolicy.VOLATILE
)

# Im Publisher Node:
self.__pub = self.create_publisher(Counter, "/counter_topic_name", my_qos)

# Im Subscriber Node:
self.__sub = self.create_subscription(Counter, "/counter_topic_name", self.counter_callback, my_qos)

6. ROS 2 TF2 (Transformationen & Frames)

TF2 verfolgt die relativen Positionen und Orientierungen (Koordinatensysteme) aller Roboterteile.

  • Frames: Koordinatensysteme (z. B. map, odom, base_link, laser_frame).
  • TF2: Berechnet automatisch relative Koordinaten.
  • tf_static: Für Frames, die sich nicht zueinander bewegen. Spart Bandbreite.
  • Broadcaster: Sendet Transformationen an das System.

Kommandozeilen-Beispiel (Statischen Frame über CLI senden):

ros2 run tf2_ros static_transform_publisher 0.1 0 0.2 0 0 0 base_link laser_frame

Code-Beispiel: Statischer TF Broadcaster (Python)

import rclpy
from rclpy.node import Node
from tf2_ros.static_transform_broadcaster import StaticTransformBroadcaster
from geometry_msgs.msg import TransformStamped

class StaticFramePublisher(Node):
    def __init__(self):
        super().__init__('static_tf_broadcaster')
        self._broadcaster = StaticTransformBroadcaster(self)
        
        t = TransformStamped()
        t.header.stamp = self.get_clock().now().to_msg()
        t.header.frame_id = 'base_link'    # Ursprung
        t.child_frame_id = 'laser_frame'   # Ziel
        
        # Translation (Verschiebung in Metern)
        t.transform.translation.x = 0.1
        t.transform.translation.y = 0.0
        t.transform.translation.z = 0.2
        
        # Rotation (Quaternion)
        t.transform.rotation.w = 1.0 # Keine Drehung
        
        self._broadcaster.sendTransform(t)

def main():
    rclpy.init()
    node = StaticFramePublisher()
    rclpy.spin(node)
    rclpy.shutdown()

7. ROS 2 Control (ros2_control)

Ein Framework zur echtzeitfähigen Steuerung von Robotern. Es trennt Hardware und abstrakte Logik.

  • Hardware Interfaces: Die Treiber-Schicht. Kommuniziert direkt mit echten Motoren/Sensoren.
  • Controller: Die Logik-Schicht. (z.B. diff_drive_controller). Nehmen Befehle (cmd_vel) an und wandeln sie um.
  • Controller Manager: Das Herzstück. Verwaltet Hardware Interfaces und Controller.

class TurtleController(Node):
    def __init__(self):
        super().__init__('turtle_controller')
        self.is_moving = False
        self.publisher_ = self.create_publisher(Twist, "/turtle1/cmd_vel", 10)
        self.timer = self.create_timer(0.1, self.timer_callback)
        self.srv_start = self.create_service(Trigger, '/start', self.start_callback)
        self.srv_stop = self.create_service(Trigger, '/stop', self.stop_callback)

    def start_callback(self, request, response):
        self.is_moving = True
        response.success = True
        return response

    def stop_callback(self, request, response):
        self.is_moving = False
        stop_msg = Twist()
        self.publisher_.publish(stop_msg)
        response.success = True
        return response

    def timer_callback(self):
        if self.is_moving:
            msg = Twist()
            msg.linear.x, msg.angular.z = 1.0, 1.0
            self.publisher_.publish(msg)

Code-Beispiel zu Aufgabe 8: Erweitertes Launch-File mit Argumenten (start.launch.py)
Dieses Skript zeigt LaunchDescription, Deklaration von Argumenten, OpaqueFunction für Laufzeitlogik und IfCondition für das bedingte Starten von Nodes (wie RViz). 

from launch import LaunchDescription
from launch.actions import LogInfo, DeclareLaunchArgument, OpaqueFunction
from launch.substitutions import LaunchConfiguration, PathJoinSubstitution
from launch.conditions import IfCondition
from launch_ros.actions import Node
from launch_ros.substitutions import FindPackageShare

def launch_setup(context, *args, **kwargs):
    """
    Diese Funktion wird zur LAUFZEIT ausgeführt. 
    Hier kann man auf den Wert von LaunchConfigurations zugreifen.
    """
    vis_value = LaunchConfiguration("visualization").perform(context)
    
    if vis_value.lower() == "true":
        return [
            LogInfo(msg="Visualisierung ist aktiviert, RViz wird gestartet..."),
            LogInfo(msg=" ")
        ]
    else:
        return [
            LogInfo(msg="Visualisierung ist deaktiviert."),
            LogInfo(msg=" ")  
        ]

def generate_launch_description():
    ld = LaunchDescription()

    # 1. Deklarationen
    ld.add_action(LogInfo(msg="Hello from the launch-System..."))

    vis_arg = DeclareLaunchArgument(
        "visualization", 
        default_value="false",
        description="Start Rviz (true/false)"
    )
    ld.add_action(vis_arg)

    # 2. Nodes (Statisch)
    turtlesim_node = Node(
        package="turtlesim",
        executable="turtlesim_node",
        name="marius"        
    )
    ld.add_action(turtlesim_node)

    draw_circle_node = Node(
        package="python_basics",
        executable="draw_circle",
        name="rene"
    )
    ld.add_action(draw_circle_node)

    # 3. OpaqueFunction (Laufzeit-Funktion aufrufen)
    ld.add_action(OpaqueFunction(function=launch_setup))

    # 4. RViz Pfad-Konstruktion
    rviz_config_path = PathJoinSubstitution([
        FindPackageShare("python_basics"),
        "rviz",
        "default.rviz"
    ])

    # 5. RViz Node mit IfCondition
    rviz_node = Node(
        package="rviz2",
        executable="rviz2",
        condition=IfCondition(LaunchConfiguration("visualization")),
        arguments=["-d", rviz_config_path]
    )
    ld.add_action(rviz_node)

    return ld
