Dashboard Web-Anbindung (Ports)

Warum zwei verschiedene Ports?

Das ROS 2 Dashboard nutzt zwei unterschiedliche Ports (8080 und 9090), um sicher und performant zu funktionieren. Hierbei handelt es sich um eine gängige Architektur für moderne Web-Anwendungen, die Echtzeitdaten verarbeiten:

  • Port 8080 (Der Datei-Lieferant)
    Gestartet durch python3 -m http.server 8080.
    Das ist ein statischer Webserver. Er lädt lediglich das visuelle Grundgerüst des Dashboards (die nackten HTML-, CSS- und JavaScript-Dateien) in deinen Browser. Er hat keinerlei Ahnung von ROS 2, Nodes oder dem Roboter.
  • Port 9090 (Der Live-Daten-Tunnel)
    Gestartet durch ros2 launch rosbridge_server rosbridge_websocket_launch.xml.
    Sobald dein Browser das Dashboard von Port 8080 geladen hat, führt er den JavaScript-Code aus. Dieser Code baut dann im Hintergrund eine dauerhafte WebSocket-Verbindung (ws://localhost:9090) zur ROS-Bridge auf. Über diesen Tunnel strömen ab sofort alle Live-Informationen (Topics, Frequenzen, Ping) in ständiger Bewegung hin und her.

Fazit: Du öffnest im Browser immer die :8080, um das Dashboard überhaupt erst "anzusehen". Das Dashboard holt sich dann seine "Seele" (die Livedaten) still und heimlich über die :9090.
