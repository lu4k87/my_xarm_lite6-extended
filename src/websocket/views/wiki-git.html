Wiki: Git Online Repository (SSH Setup)

Lokales Repository (Workspace) mit GitHub verbinden

Die beste Art, deinen Workspace (~/dev_ws) zu sichern und mit Antigravity an mehreren PCs zu arbeiten, ist die Verwendung von SSH. Dies verhindert ständige Passwortabfragen.

Schritt 1: SSH-Key generieren & bei GitHub hinterlegen

Um ohne Passwort zu arbeiten, muss für jeden PC ein SSH-Schlüssel erstellt werden:

# 1. Key generieren (E-Mail anpassen)
ssh-keygen -t ed25519 -C "antigravity-pc"

# 2. Public Key anzeigen und bei GitHub (Settings -> SSH Keys) einfügen
cat ~/.ssh/id_ed25519.pub

Schritt 2: Das lokale Git direkt im Workspace verknüpfen

Wir verknüpfen das Repo direkt mit ~/dev_ws, damit auch Dateien wie start.sh gesichert werden:

cd ~/dev_ws
git init
git remote add origin git@github.com:Username/Repo-Name.git
git branch -M main
git push -u origin main

Die wichtigsten Git-Befehle (Push & Pull)

Führe diese Befehle immer aus dem Hauptverzeichnis (~/dev_ws) aus. Deine Ordnerstruktur (wie src/) bleibt dabei automatisch exakt intakt!

PULL (Vom Server laden)

cd ~/dev_ws
git pull origin main

PUSH (Zum Server senden)

cd ~/dev_ws
git add -A
git commit -m "Hier kurze Beschreibung eintragen"
git push origin main

Zusammenarbeit mit Antigravity:

Sobald SSH steht, kann Antigravity alles für dich erledigen. Nutze einfach Befehle wie:

  • "Pushe meine aktuellen Änderungen an der start.sh"
  • "Hole den aktuellen Stand von GitHub (Pull)"
  • "Richte GitHub via SSH auf meinem 2. PC ein"

Wichtiger Hinweis:

Achte darauf, dass vertrauliche Dateien oder große Binärdaten in der .gitignore stehen, bevor du den ersten Push machst!
